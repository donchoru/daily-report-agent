Background

개발 배경 및 목적

반도체 FAB 물류 현장에서 발생하는 수작업 일보의 작성/분석 과정에서 반복적인 비효율과 정보 전달 지연 문제를 해결하기 위해 시작되었습니다.

🔴

As-Is (현재 문제점)

  • 수동 일보 작성
    교대조별로 Excel/수기로 일보를 작성. 평균 30~60분/일 소요
  • 데이터 불일치
    작성자마다 포맷/기준이 달라 시계열 비교 어려움. 전일비교 수동 계산 오류 빈발
  • 분석 지연
    핵심 이슈 파악에 추가 시간 필요. 임원/실무자별 맞춤 리포트 별도 작성
  • 이력 관리 부재
    과거 일보 검색/비교 어려움. 트렌드 분석이 사실상 불가능
🟢

To-Be (목표 상태)

  • 자동 일보 생성
    데이터 수집부터 차트 렌더링까지 1분 이내 자동 완료
  • 표준화된 포맷
    13종 차트 + KPI + Worst 10으로 일관된 일보 품질 보장
  • AI 즉시 분석
    핵심이슈 3가지 자동 추출. 역할별 맞춤 요약. 대화형 Q&A
  • 트렌드 가시화
    월/주/일별 트렌드 자동 비교. 전일대비 악화 지표 즉시 알림

핵심 개발 방향

⚡

속도 우선

PoC 빠른 검증을 위해 NiceGUI + Matplotlib 조합으로 최소 코드, 최대 기능 구현. 별도 프론트엔드 없이 Python Only.

🧠

AI 네이티브

단순 자동화가 아닌 AI가 데이터를 이해하고 인사이트를 제공하는 구조. OCR, 분석, 요약, 챗봇 4단계 파이프라인.

🔄

확장 가능

데이터 소스만 교체하면 실서비스 전환 가능한 구조. 프롬프트 템플릿 분리로 도메인 커스터마이징 용이.

Overview

프로젝트 개요

반도체 FAB 물류 현장에서 발생하는 수작업(수동 반송, 우선순위 변경, 기준정보 변경 등)을 체계적으로 관리하고, AI가 핵심 이슈를 자동 분석하는 시스템입니다.

📄

일보 자동 생성

날짜 범위를 지정하면 수작업 현황, 반송 트렌드, TAT, Worst 10, KPI 등을 포함한 고품질 PNG 일보 이미지를 자동 생성합니다. 월/주/일별 트렌드 차트 13종이 포함됩니다.

🧠

AI 분석 Agent

Gemini 2.0 Flash Vision으로 일보 이미지를 OCR하고, 핵심 이슈 3가지를 자동 추출합니다. 임원용/실무자용 맞춤 요약과 대화형 Q&A 챗봇을 제공합니다.

📊

13종 트렌드 차트

수작업 현황, 수동반송율, 우선순위 변경, LOT 우선순위, 기준정보 변경, TAT, 카세트 반송 등 월/주/일별 트렌드

💬

대화형 Q&A

일보 데이터를 컨텍스트로 자유로운 질문 가능. 추천 질문 버튼으로 빠른 인사이트 도출

🎯

역할별 맞춤 요약

임원용(15초 핵심 브리핑)과 실무자용(즉시 조치 Top 3, TAT 병목) 두 가지 요약 제공

🧠 AI 분석 Agent (Gemini 2.0 Flash)

5개의 전문 프롬프트를 활용한 멀티 스텝 AI 분석 파이프라인입니다.

Step 1. OCR 추출

Gemini Vision으로 일보 이미지를 읽어 JSON으로 구조화. 차트 값은 바 높이/파이 비율로 추정.

Step 2. 핵심이슈 추출

전일비교 데이터에서 핵심 이슈 3가지 추출. 긴급도별 색상 코딩 (빨강/노랑/초록).

Step 3. 역할별 맞춤 요약

임원용: 15초 핵심 브리핑, 월간 달성률, 악화 알림. 실무자용: 즉시 조치 Top 3, TAT 병목 구간.

Step 4. 대화형 챗봇

일보 데이터를 컨텍스트로 자유 질문. 대화 히스토리 유지. 추천 질문 4종 제공.

🎨 Premium Glassmorphism UI

다크 테마 기반의 Glassmorphism 디자인 시스템을 적용했습니다. Animated Mesh Gradient 배경, Floating Particles 효과, Glass 카드, Progress Bar 애니메이션, Fade-up 전환 등 프리미엄 UX를 제공합니다. NiceGUI + Quasar 프레임워크 기반으로 반응형 레이아웃을 지원합니다.

Current Process

현재 동작 프로세스

두 가지 메인 워크플로우가 웹 앱에서 동작합니다.

Workflow 1: 일보 생성기 / 페이지

1

날짜/개수 입력

시작일(YYYY-MM-DD)과 생성 개수(1~90일)를 입력합니다.

2

데이터 생성

날짜 시드 기반으로 10개 카테고리의 샘플 데이터를 생성합니다.

3

차트 렌더링

Matplotlib로 22x72 인치 캔버스에 13종 차트 + 테이블을 렌더링합니다.

4

결과 출력

PNG 저장, 통계 요약(총 수작업/평균 달성률), 카드형 결과 목록을 표시합니다.

Workflow 2: AI 분석 Agent /ai 페이지

1

이미지 업로드

일보 PNG/JPG를 업로드하거나, "샘플 데이터로 테스트" 버튼으로 바로 시작합니다.

2

OCR + 구조화

Gemini Vision이 이미지를 읽어 JSON으로 구조화합니다. 차트 값도 추정 추출합니다.

3

AI 분석

핵심이슈 3가지 추출 + 임원용/실무자용 맞춤 요약을 병렬 생성합니다.

4

대화형 Q&A

분석 결과를 컨텍스트로 자유 질문. 추천 질문 버튼 4종으로 빠른 인사이트를 도출합니다.

Challenges & Risks

도전 과제 및 리스크

프로덕션 전환 과정에서 예상되는 주요 도전 과제와 대응 방안입니다.

HIGH RISK

실데이터 연동 복잡도

MES/WMS 시스템별로 데이터 포맷, 접근 방식, 권한 체계가 상이. 데이터 정합성 검증이 핵심.

대응 방안: 데이터 어댑터 패턴으로 소스별 추상화. 단계적 연동 (1개 FAB → 전체 확장).
HIGH RISK

AI 분석 품질 일관성

LLM 특성상 동일 데이터에 대해 다른 분석 결과를 생성할 수 있음. 핵심 수치 인용 정확도 관리 필요.

대응 방안: 프롬프트 엔지니어링 지속 개선. 사전 계산된 비교 데이터를 컨텍스트에 주입. 결과 검증 레이어 추가.
MED RISK

사내망 환경 제약

SSL 인증서, 프록시, 방화벽 등 사내망 환경에서 외부 API (Gemini) 호출 안정성.

대응 방안: SSL SECLEVEL=0 우회 적용 완료. 재시도 로직 + 타임아웃 설정. 향후 온프레미스 모델 검토.
MED RISK

사용자 수용성

현장 작업자의 기존 업무 방식 변경 저항. AI 분석 결과에 대한 신뢰도 확보 시간 필요.

대응 방안: 기존 일보와 병행 운영. 현장 피드백 반영 주기적 개선. 사용법 교육 및 챔피언 사용자 육성.
LOW RISK

API 비용 관리

Gemini API 호출량 증가에 따른 비용. 특히 OCR + 분석 + 요약 + 챗봇 4단계 호출.

대응 방안: Gemini 2.0 Flash는 저비용 모델. 결과 캐싱으로 중복 호출 방지. 일일 사용량 모니터링.
LOW RISK

데이터 보안

일보 데이터에 생산량, 설비 상태 등 민감 정보 포함. 외부 API로 전송 시 보안 이슈.

대응 방안: 데이터 마스킹/익명화 처리. 보안팀 검토 후 승인. 장기적으로 온프레미스 LLM 전환 검토.
