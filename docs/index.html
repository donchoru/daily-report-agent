Background

개발 배경 및 목적

반도체 FAB 물류 현장에서는 매일 수작업 일보가 발행됩니다. 하지만 일보를 읽고 핵심을 파악하는 데 많은 시간이 소요되고, 사람마다 해석이 달라 정보 전달에 왜곡이 발생합니다.

🔴

As-Is (현재 문제점)

  • 일보 해석에 시간 과다 소요
    차트 13종 + 테이블 + KPI를 사람이 읽고 핵심을 파악하는 데 30분 이상 소요
  • 보고 대상별 재가공 필요
    동일 일보를 임원/실무자/교대조에게 각각 다른 포맷으로 요약해서 전달해야 함
  • 핵심 이슈 누락 위험
    데이터가 많아 전일 대비 악화 지표, 이상 패턴을 놓치는 경우 빈발
  • 추가 질의 불가
    일보에 대한 추가 궁금증이 있어도 데이터를 다시 꺼내서 확인해야 함
🟢

To-Be (AI Agent 도입 후)

  • 일보 업로드 즉시 핵심 파악
    AI가 일보 이미지를 OCR로 읽고 10초 이내 핵심이슈 3가지 자동 추출
  • 역할별 맞춤 요약 자동 생성
    임원용(15초 브리핑) / 실무자용(즉시 조치 Top 3) 동시 생성
  • 전일비교 자동 분석
    모든 지표의 전일 대비 변화를 자동 계산, 악화 지표 즉시 알림
  • 대화형 Q&A
    "Worst 10 중 즉시 조치 필요한 건?" 같은 질문에 AI가 즉시 답변

핵심 개발 방향

👁

읽는 AI

기존 일보 이미지를 Gemini Vision OCR로 읽어서 구조화된 데이터로 변환. 사람이 다시 입력할 필요 없음.

🧠

분석하는 AI

단순 요약이 아닌, 전일비교/악화지표/병목구간까지 분석. 역할별 맞춤 인사이트 제공.

💬

대화하는 AI

일보 데이터 기반 자유 질의응답. 추가 궁금증을 바로 해결. 대화 히스토리 유지.

AI Analysis Pipeline

AI 분석 파이프라인

일보 이미지 한 장을 업로드하면, 4단계 AI 파이프라인이 자동으로 핵심 인사이트를 추출합니다.

📄 기존 일보
PNG/JPG 업로드
→
👁 Gemini Vision
OCR → JSON 구조화
→
🧠 AI 분석
이슈 추출 + 요약
→
💬 인사이트
요약 + 챗봇 Q&A
1

OCR 구조화

Gemini 2.0 Flash Vision이 일보 이미지를 읽어 차트/테이블/KPI 등 모든 데이터를 구조화된 JSON으로 변환합니다.

추출 대상: 보고일/근무조/작성자, 핵심 KPI, 트렌드 수치 (차트 바 높이 추정), Worst 10 테이블, 주요 이슈 텍스트
2

핵심이슈 자동 추출

전일비교 데이터를 기반으로 핵심 이슈 3가지를 추출합니다. 긴급도별 색상 코딩으로 한눈에 파악 가능합니다.

🔴 긴급   🟠 주의   🟢 양호
각 이슈에 수치 근거 + 추천 조치 포함
3

역할별 맞춤 요약

동일 일보를 두 가지 관점으로 동시에 요약합니다. 각 역할에 필요한 정보만 선별하여 최적화된 보고를 생성합니다.

💼 임원용
15초 핵심 브리핑, 월간 달성률, 악화 알림, 의사결정 포인트
🛠 실무자용
즉시 조치 Top 3, TAT 병목 구간, 수동반송 현황, 부서-공정 지정
4

대화형 Q&A 챗봇

일보 데이터를 컨텍스트로 자유 질문이 가능합니다. 대화 히스토리가 유지되어 연속적인 분석이 가능합니다.

추천 질문 예시:
"오늘 가장 큰 이슈가 뭐야?" • "TAT 병목 구간은 어디야?" • "Worst 10 중 즉시 조치가 필요한 건?" • "수동반송율 추이는 어때?"

📝 프롬프트 설계 (5종)

각 분석 단계에 최적화된 전문 프롬프트를 별도 파일로 관리합니다. 도메인 지식(반도체 물류)을 프롬프트에 내장하여 분석 품질을 보장합니다.

프롬프트 역할 핵심 지시사항
ocr_extract.txt OCR 구조화 이미지의 모든 데이터를 JSON으로 변환. 차트 값은 바 높이/파이 비율로 추정
key_issues.txt 핵심이슈 추출 정확히 3가지 이슈 추출. 수치 근거 필수. 긴급도별 색상 코딩
executive.txt 임원용 요약 15초 핵심 브리핑. 표 형식. 월간 달성률. 악화 지표 알림
operator.txt 실무자용 요약 즉시 조치 Top 3. TAT 병목. 부서-공정 지정. 최대 2줄
chat_system.txt 챗봇 시스템 데이터 기반 답변 only. 추측 금지. 한국어 응답. 실무적 조언

Challenges & Risks

도전 과제 및 리스크

실서비스 전환 시 예상되는 주요 도전 과제와 대응 방안입니다.

HIGH

OCR 정확도

일보 이미지 품질(해상도, 글꼴, 레이아웃)에 따라 OCR 정확도가 달라질 수 있음. 특히 차트 수치 추정에 오차 발생 가능.

대응: 실제 일보로 OCR 정확도 측정 → 프롬프트 튜닝. 사용자 수정 에디터 제공. 향후 데이터 직접 연동으로 OCR 의존 제거.
HIGH

AI 분석 일관성

LLM 특성상 동일 데이터에 다른 분석을 생성할 수 있음. 핵심 수치 인용 정확도 관리 필요.

대응: 사전 계산된 비교 데이터를 컨텍스트에 주입 (이미 적용). 프롬프트에 "추측 금지, 수치 근거 필수" 명시.
MED

사내망 환경 제약

SSL/프록시/방화벽 환경에서 외부 Gemini API 호출 안정성.

대응: SSL SECLEVEL=0 우회 적용 완료. 재시도/타임아웃 설정. 장기: 온프레미스 LLM 전환 검토.
MED

데이터 보안

일보 이미지에 생산량, 설비 상태 등 민감 정보 포함. 외부 API 전송 시 보안 검토 필요.

대응: 보안팀 검토 후 승인. 데이터 마스킹 처리. 장기: 온프레미스 모델로 전환.
LOW

사용자 수용성

기존 방식 변경에 대한 저항. AI 분석 결과에 대한 신뢰 확보 시간.

대응: 기존 방식과 병행 운영. 현장 피드백 반영. 챔피언 사용자 육성.
LOW

API 비용

일보당 OCR + 이슈 + 요약x2 + 챗봇 = 최소 4~5회 API 호출.

대응: Gemini 2.0 Flash는 저비용 모델. 결과 캐싱. 일일 사용량 모니터링.
